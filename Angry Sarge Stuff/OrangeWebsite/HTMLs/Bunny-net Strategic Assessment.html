4. Performance & Latency Analytics

Quantifying the "Speed of Light" penalty vs. the "Edge Caching" advantage.

The Challenge: Physics. Hosting origin files in Slovenia introduces a baseline latency of ~130ms for a user in Texas.

The Solution: Bunny.net's 119+ PoPs (Points of Presence). When a US user requests a video:

  1. Request 1 (Miss): User hits Dallas PoP. File not there. Dallas fetches from Slovenia (140ms delay).
  2. Cache Lock: Dallas PoP stores the file.
  3. Request 2+ (Hit): Next user hits Dallas PoP. File served instantly (20ms).

Result: 95% of traffic never touches the Slovenian origin server after the first hit.
